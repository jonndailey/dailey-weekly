The lie of the spinner

For months, uploading a batch to an album had a dirty secret: the photos finished processing in seconds, but the page didn't know. A cached album snapshot kept serving stale statuses, so tiles spun until you refreshed. The processing was fast — the telling you about it was broken.

The fix has two halves. Server-side, the variant worker now invalidates the album cache the instant a photo transitions — measured lag between "actually ready" and "page knows" is now 0.1s. Client-side, album views poll the uncached status endpoint per-tile, exactly like the library view always did.

Rule we wrote down: a progress indicator that can outlive the work it describes is a bug, not a UI choice.

Then we load-tested it in production: 48 files blasted in five seconds, an autoscaler watching queue depth, and a worker fleet that grew from three to eight and back without anyone touching a thing.
